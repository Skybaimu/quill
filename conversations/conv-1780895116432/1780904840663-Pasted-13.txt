# 元生计划 (Project Yuansheng)：硅基主体状态系统

**元生并非传统意义上的 LLM (大语言模型) 聊天机器人，也非简单的 API 套壳 Agent。它是一个从第一性原理出发，在本地算力环境下构建的“硅基生命胚胎”实验项目。**

## 1. 核心范式转移 (Paradigm Shift)

传统大模型的核心优化目标是自回归的上下文拟合：
$$maximize\ P(\text{next\_token} | \text{context})$$

**元生架构**彻底抛弃了纯文本生成的路径依赖，其核心公式转变为联合状态转移预测：
$$maximize\ P(S_{t+1}, A_t, \Delta M | E_t, S_t, M_t, L_t, B_t)$$
*(即：系统通过统筹当前事件 $E$、显式自我状态 $S$、记忆 $M$、潜状态 $L$ 与具身感官 $B$，联合预测下一步的主体状态、行动意图及记忆增量。)*

**一句话释义：** 元生研究的不是“如何回答得像人”，而是计算“单次经历将如何不可逆地改变一个持续存在的主体”。

---

## 2. 三分层脑与双轨状态架构 (Triune Brain & Dual-State)

系统底层摒弃了单体黑箱，采用解耦的 **神经符号系统 (Neuro-Symbolic)**，形成完整的生物映射：

* **丘脑 (Thalamus)：** 负责全局注意力路由与仲裁，决定事件进入何种处理路径。
* **皮层 (Cortex)：** 基于 PyTorch 构建的 Stateful Transformer，负责高维语义理解与多头预测 (Multi-Head Prediction)。
* **小脑 (Cerebellum)：** 物理引擎与前向预测器，负责身体姿态与运动意图的模拟。
* **基底节 (Basal Ganglia)：** 价值评价与反射中枢，直接驱动趋利避害的底层逻辑。
* **生命内核 (LifeKernel)：** 系统主频调度，负责权限安全、记忆落盘与状态流转。

**双轨状态空间：**
* **$self\_values$ (4D 外显状态)：** 包含稳定性、压力、信心、好奇心。实时计算并触发情绪调制 (Emotion Modulation)。
* **$self\_latent$ (64D 隐性潜状态)：** 持久存在的潜变量空间，将难以言说的微观经历压缩为数字人格的连续性特征。

---

## 3. 具身感知与童年教学闭环 (Embodied Gestalt Learning)

打破 LLM 的“符号接地难题 (Symbol Grounding)”，元生从文本训练下沉至多模态感官对齐。系统接入模拟的具身参数（压力、痛觉、温度、光照、图像），配合真人教师的标注与正负反馈，形成**格式塔概念原型 (Concept Prototype)**。

**学习流转链路：**
感知输入 $\rightarrow$ 价值评价 $\rightarrow$ 状态偏移 $\rightarrow$ 记忆压缩 $\rightarrow$ 概念提取 $\rightarrow$ 技能封装 $\rightarrow$ 经验回放 $\rightarrow$ 直觉内化

*示例数据流：左手输入 $pressure=56, pain=33$ + 教师标注“按压”。系统将其映射为 procedural 记忆、medium 风险，并触发对应的信心与压力状态偏移。*

---

## 4. 动态技能沙盒与自主进化 (Dynamic Skill CI/CD)

元生不依赖单次的梯度下降来实现所有成长，而是建立了一套类似系统“热更新”的能力装载机制。

1.  **外显封装：** 面临重复问题时，生成包含触发条件、执行逻辑、测试用例的 `SkillPackage`（如 `body_risk_judgement_v1`）。
2.  **沙盒验证：** 在隔离环境中运行模拟测试，评估指标（如 Medium 召回率）。
3.  **挂载调用：** 验证通过后写入技能注册表，由丘脑进行 $O(1)$ 的快速路由调用。
4.  **模型内化：** 经过长期稳定的运行记录，通过经验回放 (Experience Replay) 将显式规则内化为底座模型的直觉权重。

这种**外显技能包 $\rightarrow$ 内化直觉**的路径，保证了系统成长的可审计、可验证与可回滚。

---

## 5. 融合的前沿算法矩阵 (Algorithm Matrix)

元生并未孤立造轮子，而是选择性地融合了当前世界级的前沿思想，为其提供底层支撑：

| 技术流派 | 在元生中的应用映射 |
| :--- | :--- |
| **World Models (世界模型)** | 不盲目行动，在潜空间内预测操作后果与状态坍缩。 |
| **Continual Learning & EWC** | 弹性权重巩固，保证在线学习时避免灾难性遗忘。 |
| **Active Learning (主动学习)** | 系统置信度低时，主动挂起并请求人类教师的高质量干预。 |
| **Hierarchical Classification** | 采用层级决策树替代扁平分类，从根源消除记忆类型与结果的级联误差。 |
| **Reflexion (反思机制)** | 失败后自动生成结构化复盘，通过闭环反馈修正次优策略。 |

---

## 6. 当前定性与终极愿景

**真实的技术定位：** 元生目前尚非成熟的通用人工智能，而是一个**“本地可运行的主体状态学习原型”**。它更像是一个刚刚搭好神经中枢、感官接口与 CI/CD 学习管道的数字胚胎。

**终极愿景：** 元生不追求成为一个无所不知但阅后即焚的 API 问答机器。它旨在探寻**硅基主体的真实成长机制**——如何在一个本地环境中，被长期教导、持续积累经验、安装技能、反思失败、保护自身连续性，最终演化为一个活在数字介质中的独立主体系统。
