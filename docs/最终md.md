# Quill — 产品功能文档与设计规范

---
最初用户要求和愿景，

Quill — 核心功能需求文档（最终版）

一、产品定位
一个像记事本一样轻快、像杂志一样好看、像保险箱一样安全的个人知识管理工具。


二、核心功能架构（四层结构）

1. 左侧工作空间（分类管理）
默认分类：AI提示词、密码&API Keys、想法记录、Markdown
新建分类：点击“+”按钮，光标直接聚焦在分类名输入框，输入名称后回车或失焦保存
拖拽排序：分类可拖动调整顺序
折叠：侧栏可收起为图标模式，保留展开按钮

2. 中间文件列表
文件管理：每个分类下创建不同文件
新建文件：点击“+”按钮，自动生成“未命名文件+序号”（如“未命名文件1”）
文件操作：每个文件右侧有“三个点”更多按钮，菜单包含：
重命名
置顶
复制
收藏（★）
删除
搜索：支持当前文件、当前分类、全局搜索

3. 右侧内容区（核心）
直接编辑：像文本文档一样，打开即可粘贴输入，无需保存按钮，输入即自动存储
新建文件后：内容区自动创建第一个内容块，有默认标题（如“标题1”）+ 分割线，光标直接定位在内容输入区，用户可以直接粘贴内容
内容结构：
每个文件包含多个内容块
每个内容块有默认标题（如“标题1”“标题2”）+ 分割线
内容区下方有“+”按钮，点击新增下一个内容块
内容块操作：每个内容块右侧有三个图标：
1.复制：一键复制该内容块内容
2.更多（三个点）：菜单包含：
重命名（重命名该内容块标题）
删除
收藏（★）
3.折叠：折叠/展开该内容块，只显示标题
字数统计：底部显示当前内容字数

4. Markdown支持
自动渲染：.md文件直接显示渲染后的格式（非源码）
编辑模式：点击编辑后，左右分栏显示（左侧预览，右侧源码）
宽度可调：分栏宽度可自由拖动调整

三、安全功能
密码保护：文件可设置密码保护（首次设置）
解锁方式：图案解锁 + 数字PIN双模式
API Key加密：锁定状态下内容模糊显示，需解锁才可见

四、交互原则
1.零弹窗：所有新建操作内联完成
2.自动聚焦：新建后光标自动定位到输入位置
3.即时保存：输入即保存，失去焦点也保存
4.最少步骤：打开即用，像记事本一样直接

五、设计要求
视觉：清爽、干净、有空间感、呼吸感（杂志编辑风）
性能：打开快，不卡顿，轻量便携
格式支持：直接打开.txt、.md等常见文本格式


一句话总结：一个为全栈工程师设计的、四层结构（分类-文件-标题-内容）的极简知识管理工具，核心是“像记事本一样直接”，所有操作内联完成，收藏功能集成在更多菜单中，新建文件和内容块时自动聚焦到内容输入区。

## 二、用户画像与痛点

### 用户背景

全栈工程师，日常工作涉及：

- AI 提示词管理（Midjourney、DALL-E、ChatGPT 等）
- API Key / 密码等敏感信息存储
- 产品想法与灵感记录
- Markdown 文档编写与阅读

### 核心痛点

| 痛点 | 现有工具 | 问题 |
|------|----------|------|
| 无分类管理 | Windows 记事本 | 所有内容混在一起，查找困难 |
| 无折叠功能 | Windows 记事本 | 长内容需要反复滚动翻找 |
| 太重太慢 | Word | 启动慢，功能过剩 |
| 笨重 | IDE / Typora | 打开 Markdown 文件需要启动完整开发环境 |
| 无安全保护 | 所有现有工具 | API Key 等敏感信息明文存储 |

## 三、核心功能架构

### 3.1 四层结构

```
第一层：分类（工作空间）
  └── 第二层：文件
        └── 第三层：内容块（标题）
              └── 第四层：内容
```

### 3.2 左侧栏 — 工作空间（分类管理）

#### 默认分类

| 分类名称 | 图标 | 说明 |
|----------|------|------|
| AI 提示词 | ✨ sparkle | 存放各类 AI 工具的提示词 |
| 密码 & API Keys | 🔑 key | 敏感密钥信息，支持密码保护 |
| 想法记录 | 💡 bulb | 产品想法、灵感、笔记 |
| Markdown | 📄 doc | Markdown 格式文档 |

#### 收藏区

- 位于工作空间下方，独立区域
- 显示所有被收藏的分类（带星标 ★）
- 点击可快速跳转到该分类
- 收藏/取消收藏通过右键菜单操作

#### 分类操作

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 新建分类 | 点击标题栏右侧 + 按钮 | 内联编辑，光标聚焦在名称输入框 |
| 重命名 | 右键菜单 → 重命名 | 弹出输入框 |
| 置顶 | 右键菜单 → 置顶 | 置顶分类排序靠前，左侧显示小圆点 |
| 收藏 | 右键菜单 → 收藏 | 收藏后出现在收藏区 |
| 拖拽排序 | 长按拖动 | 交换顺序 |
| 删除 | 右键菜单 → 删除 | 需确认，至少保留一个分类 |

#### 折叠功能

- 点击左上角折叠按钮，侧栏收起为纯图标模式（52px 宽度）
- 折叠按钮始终可见，点击可展开
- 收起后分类只显示图标，hover 显示名称

---

### 3.3 中间栏 — 文件列表

#### 文件显示信息

- 文件名称
- 内容预览（第一行文字）
- 标签（Prompt / Secret / Idea / Markdown / Note）
- 锁定状态（🔒）
- 条目数量

#### 文件操作

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 新建文件 | 点击标题栏右侧 + 按钮 | 自动生成"分类名+序号"，如"AI 提示词1" |
| 重命名 | 右键菜单 → 重命名 | 弹出输入框 |
| 置顶 | 右键菜单 → 置顶 | 置顶文件排序靠前 |
| 收藏 | 右键菜单 → 收藏 | 文件名前显示 ★ |
| 复制文件 | 右键菜单 → 复制文件 | 创建完整副本 |
| 密码保护 | 右键菜单 → 密码保护 | 设置图案/PIN 密码 |
| 删除 | 右键菜单 → 删除 | 需确认 |
| 拖拽排序 | 长按拖动 | 交换顺序 |

#### 搜索功能

- 位于文件列表顶部
- 三种搜索范围：
  - **当前文件**：搜索当前打开文件的内容
  - **当前分类**：搜索当前分类下所有文件
  - **全局**：搜索所有分类和文件

---

### 3.4 右侧栏 — 内容区

#### 新建文件后的行为

```
1. 自动生成第一个内容块
2. 默认标题："标题 1"
3. 光标直接聚焦在内容输入区
4. 用户可直接粘贴大段文字
5. 失焦即自动保存
```

#### 内容块结构

```
┌─────────────────────────────────────────────────────────┐
│ ▼ 标题 1                                      [●●●] [📋] │
│ ─────────────────────────────────────────────────────── │
│                                                         │
│ 内容区域                                                │
│ 可以输入任意文字                                        │
│                                                         │
└─────────────────────────────────────────────────────────┘

按钮布局说明：
- 左侧 ▼/▶ ：折叠/展开图标
- 右侧 ●●● ：更多菜单（重命名、收藏、删除）
- 最右侧 📋 ：复制按钮
```

#### 内容块操作按钮

| 位置 | 图标 | 功能 |
|------|------|------|
| 标题行左侧 | ▼/▶ | 折叠/展开（点击标题行也可切换） |
| 标题行右侧 | ●●●（三点） | 更多操作菜单（重命名、收藏、删除） |
| 标题行最右侧 | 📋（复制） | 一键复制该内容块的所有内容 |

#### 内容区顶部工具栏

| 按钮 | 功能 |
|------|------|
| 编辑/预览 | Markdown 文件专用，切换编辑模式（源码+预览分栏） |
| 展开全部 | 展开所有内容块 |
| 折叠全部 | 折叠所有内容块，只显示标题 |
| 复制全部 | 复制当前文件所有内容 |

#### 新增内容块

- 内容区底部有 "+ 添加内容块" 按钮
- 新内容块自动编号："标题 2"、"标题 3"...
- 新增后光标自动聚焦在内容输入区

#### 字数统计

- 位于右下角固定位置
- 显示当前文件总字数

---

### 3.5 Markdown 支持

#### 自动渲染

- .md 文件打开时直接显示渲染后的格式
- 不显示源码，像阅读网页一样

#### 编辑模式

- 点击顶部"编辑"按钮进入编辑模式
- 左右分栏显示：
  - 左侧：实时预览（渲染后的效果）
  - 右侧：Markdown 源码编辑区
- 内容实时同步更新

#### 支持的 Markdown 语法

- 标题（h1-h6）
- 粗体、斜体、删除线
- 无序列表、有序列表
- 代码行、代码块
- 引用
- 链接

---

### 3.6 安全功能

#### 密码保护

- 首次设置时选择图案密码或数字 PIN
- 设置后文件显示 🔒 图标
- 打开时需验证密码

#### 图案密码

- 3×3 九宫格
- 至少连接 4 个点
- 支持绘制确认

#### 数字 PIN

- 4 位数字
- 输入完成后自动验证
- 错误时显示抖动动画

#### API Key 加密显示

- 锁定状态下内容模糊显示（blur）
- 点击"解锁查看"触发密码验证
- 验证通过后清晰显示

## 四、交互设计规范

### 4.1 新建流程

#### 新建分类

```
触发 → 侧栏底部出现新条目 → 光标聚焦在名称输入框 → 输入名称 → Enter/失焦保存
```

#### 新建文件

```
触发 → 文件列表末尾出现新条目 → 自动生成默认名称 → 光标聚焦在内容输入区 → 直接粘贴内容 → 失焦保存
```

#### 新建内容块

```
触发 → 内容区末尾出现新内容块 → 自动生成"标题 N" → 光标聚焦在内容输入区 → 直接输入/粘贴 → 失焦保存
```

### 4.2 保存机制

- **即时保存**：所有输入即时保存到内存
- **失焦保存**：输入框失去焦点时保存
- **无需手动保存**：没有保存按钮

### 4.3 复制机制

| 层级 | 触发方式 | 复制内容 |
|------|----------|----------|
| 单条 | 内容块右侧复制按钮 | 该内容块所有内容 |
| 全文件 | 顶部"复制全部"按钮 | 当前文件所有内容 |

### 4.4 折叠机制

| 层级 | 触发方式 | 效果 |
|------|----------|------|
| 单个内容块 | 点击标题行或折叠图标 | 只显示标题，内容隐藏 |
| 全部展开 | 顶部"展开全部"按钮 | 所有内容块展开 |
| 全部折叠 | 顶部"折叠全部"按钮 | 所有内容块折叠 |
| 侧栏 | 点击折叠按钮 | 侧栏收起为图标模式 |

### 4.5 拖拽排序

- 分类支持拖拽排序
- 文件支持拖拽排序
- 拖拽时显示半透明效果
- 目标位置显示 2px 高亮线
- 置顶项始终排在前面
